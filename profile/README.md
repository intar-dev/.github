# intar-dev

intar is a hands-on DevOps training platform built around real scenarios.
It is built by [Stefan Ruzitschka](https://icepuma.dev).

I would love to partner with companies that can donate bare metal servers to help run the scenario VMs that power the training experience. If that sounds interesting, please reach out at [marketing@intar.dev](mailto:marketing@intar.dev).

> Most of the code is currently created using Codex and GPT-5.4 with xhigh reasoning, but it is deeply tested by humans.

You can find our public repositories below.

- [`intar-cli`](https://github.com/intar-dev/intar-cli) is the local CLI for running DevOps lab scenarios in VMs, evaluating probes, and following progress in a TUI.
- [`kino`](https://github.com/intar-dev/kino) is the probe service used to check ephemeral scenario VMs during training runs.
- [`stargate`](https://github.com/intar-dev/stargate) is the gateway layer for browser and SSH terminal access into training environments.
